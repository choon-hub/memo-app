# [デプロイ #1] アプリを Vercel にデプロイし、スマホから利用できるようにする

種別: 基盤（デプロイ） / 依存 Issue: なし（ただし本番 Supabase の接続先は [デプロイ #2] で用意）

## 背景 / 目的

現在アプリはローカル開発環境（`npm run dev`）でのみ動作している。
外出先でもスマホから「1日1新」「日々のトピック」「筋トレ」を記録・参照できるよう、
アプリをホスティングサービスにデプロイして公開 URL でアクセスできるようにする。

- 利用は**個人利用のみ**（商用利用は想定しない）→ 無料枠の規約範囲内で運用できる
- 単一ユーザー・認証なしの構成のため、URL を知っていれば誰でも操作できる点は許容する
  （気になる場合の対策は「補足」を参照）

## ホスティング先の選定（2026年7月時点の無料枠）

**Vercel（Hobby プラン・無料）を推奨。** Nuxt 4 をゼロコンフィグでデプロイでき、
GitHub 連携で push のたびに自動デプロイされる。

| 候補 | 無料枠（2026年7月時点） | 備考 |
|------|--------|------|
| **Vercel Hobby**（推奨） | 帯域 100GB/月、Edge リクエスト 100万/月、Function 100万回/月 | 非商用・個人利用限定（本件は該当OK）。Nuxt 公式対応 |
| Cloudflare Workers | 静的配信の帯域は実質無制限、ビルド 500回/月、Workers リクエスト 10万/日 | 商用も可。Cloudflare は新規プロジェクトに Pages ではなく Workers を推奨する方針に移行済み。Nitro の `cloudflare_module` プリセット指定が必要 |
| Netlify Free | クレジット制 300クレジット/月：帯域 20クレジット/GB（実質 ~15GB/月）、本番デプロイ 15クレジット/回（実質 ~20回/月） | 2026年にクレジット制へ移行し無料枠が大幅縮小。クレジット枯渇で全サイト停止するため今回は非推奨 |

本アプリは Supabase をクライアントサイドから直接叩くだけ（`server/` 未使用）なので、
Vercel / Cloudflare なら無料枠で十分収まる。Netlify は無料枠縮小により候補から外す。

## やること

- [ ] GitHub リポジトリを Vercel に接続し、プロジェクトを作成（Framework Preset: Nuxt）
- [ ] Vercel の環境変数に `SUPABASE_URL` / `SUPABASE_KEY` を設定
      （値は [デプロイ #2] で作成する**本番用** Supabase プロジェクトのもの）
- [ ] main ブランチへの push で自動デプロイされることを確認
- [ ] デプロイされた URL でアプリが動作することを確認
- [ ] スマホの実機ブラウザで 3 機能（/one-new, /topics, /workout）の表示・記録を確認
- [ ] スマホ表示で崩れがあれば修正（`app/assets/global.css` 等）
- [ ] デプロイ手順と URL を `README.md`（または `docs/`）に記録

## 受け入れ条件（AC）

- [ ] 公開 URL にアクセスするとアプリが表示され、`/` が `/one-new` にリダイレクトされる
- [ ] スマホ実機から 3 機能すべてで記録の作成・一覧表示ができる
- [ ] main への push で自動的に再デプロイされる
- [ ] `.env` やシークレットがリポジトリにコミットされていない
      （環境変数は Vercel のダッシュボードでのみ管理）

## 無料枠でできること / 有料枠が必要になること（2026年7月時点）

### 無料枠（Hobby・$0）で足りること — 本 Issue はすべてここで完結する

- 個人利用のデプロイ・自動デプロイ・HTTPS・`*.vercel.app` ドメイン
- 帯域 100GB/月・Edge リクエスト 100万/月・Function 100万回/月・
  Active CPU 4時間/月 — 単一ユーザーのメモアプリでは到底届かない

### 有料枠（Pro・$20/月/人〜）が必要になるケース — 今回は不要

- 商用利用（Hobby プランは規約上、非商用・個人利用限定。広告掲載や課金導入も違反になる）
- 無料枠の上限超過時に停止させたくない場合（Hobby には従量課金オプションがなく、
  上限到達で該当機能が翌サイクル（30日）まで停止する）
- チームでの共同管理、Deployment Protection 等の保護機能

## 対象ファイル / ディレクトリ案

- `nuxt.config.ts`（必要ならプリセット調整。基本は変更不要の見込み）
- `README.md`（デプロイ手順の追記）
- `app/assets/global.css`（スマホ表示の調整が必要な場合）

## 補足

- 認証なしのため URL 漏洩＝第三者が操作可能。個人利用前提で許容するが、
  気になる場合は Vercel の Deployment Protection（Pro 機能）ではなく、
  無料でできる対策として「推測されにくい URL の維持」「将来的な Supabase Auth 導入」を検討
- 独自ドメインは Hobby プランでも設定可能（ドメイン代のみ実費）
