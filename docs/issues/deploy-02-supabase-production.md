# [デプロイ #2] Supabase 本番環境を整備し、いつでも参照できる状態にする

種別: 基盤（デプロイ） / 依存 Issue: なし（[デプロイ #1] が本 Issue の接続情報を利用する）

## 背景 / 目的

現在の Supabase プロジェクトは開発・テスト用として使っている。
デプロイ後のアプリが**本番データをいつでも参照できる**よう、本番用の Supabase 環境を整備する。

最大の論点は **無料プランでは DB が 7 日間未使用だと自動休止（pause）される**こと。
「いつでも参照できる」を満たすための休止対策を本 Issue で講じる。

## やること

- [ ] 本番用 Supabase プロジェクトを作成（無料プランは 2 プロジェクトまで作成可能。
      既存の開発用と分けるか、開発用をそのまま本番に昇格するかを決める）
- [ ] `supabase/migrations/0001_init.sql` を本番プロジェクトに適用
      （`daily_new` / `topics` / `workout_records` の 3 テーブル）
- [ ] 本番の `SUPABASE_URL` / `SUPABASE_KEY`（anon key）を取得し、
      Vercel の環境変数に設定（[デプロイ #1]）。ローカルの `.env` とは分離する
- [ ] **自動休止対策**を導入する（下記「無料枠でできること」参照）。
      推奨: GitHub Actions で定期的に DB へ軽い読み取りクエリを投げる keep-alive ワークフロー
- [ ] 開発用とテスト用のデータが本番に混ざらない運用を README に明記
      （テストは `test/mocks/supabase.ts` のモックを使うため DB 不要、という現状の整理も含む）
- [ ] RLS 未設定のまま anon key を公開する現状のリスクを README のセキュリティ節に追記
      （単一ユーザー前提で許容する判断も含めて記録）

## 受け入れ条件（AC）

- [ ] デプロイ済みアプリから本番 Supabase の 3 テーブルに読み書きできる
- [ ] マイグレーションが本番に適用済みで、`shared/types/database.ts` と実スキーマが一致している
- [ ] keep-alive（または相当の休止対策）が稼働しており、1 週間以上放置しても
      アプリからデータを参照できる
- [ ] 本番のキー・URL がリポジトリにコミットされていない

## 無料枠でできること / 有料枠が必要になること（2026年7月時点）

### 無料枠（Free・$0）でできること — 個人利用なら基本ここで完結する

- DB 500MB・ファイルストレージ 1GB・帯域（egress）5GB/月・
  Edge Function 50万回/月・アクティブ 2 プロジェクトまで・API リクエスト数は無制限
  → テキストのみの個人メモなら数年単位で余裕がある
- **7 日間 DB が未使用だと自動休止**（データ・スキーマは消えない。
  復帰は手動 or 次回アクセスから約 30 秒）
  - 無料のままの対策: GitHub Actions（cron）で数日おきに `select` を 1 回実行する
    keep-alive を回す。※休止判定は API リクエストや管理画面の閲覧ではなく
    **DB アクティビティ基準**なので、DB に届くクエリであることが必要
  - 毎日アプリを使っていれば実質休止しないが、旅行等で 1 週間空くと休止するため対策は入れておく
- 無料プランには自動バックアップ・SLA・SSO は付かない

### 有料枠（Pro・$25/月）が必要になるケース — 今回は原則不要

- 自動休止の完全な無効化（keep-alive のような回避策なしで常時稼働させたい場合）
- 自動バックアップ（Pro はデイリーバックアップ 7 日分。無料プランにはバックアップがないため、
  必要なら `pg_dump` を GitHub Actions で定期実行してエクスポートする無料の代替案で補う）
- DB 8GB 超・ストレージ 100GB・帯域 250GB/月（超過は $0.09/GB の従量課金）が必要な場合

**判断:** 個人利用・低頻度アクセスであれば、keep-alive ＋（任意で）定期 `pg_dump` により
無料枠で「いつでも参照できる」を満たせる。Pro への課金は不要と判断する。

## 補足

- keep-alive の実装例: `curl` で PostgREST エンドポイント
  （`{SUPABASE_URL}/rest/v1/daily_new?select=id&limit=1` に `apikey` ヘッダ付与）を叩く。
  シークレットは GitHub Actions の Secrets に保存する
- 休止中のプロジェクトは無料枠の「2 アクティブプロジェクト」にカウントされないため、
  開発用プロジェクトは普段休止させておいても支障ない
