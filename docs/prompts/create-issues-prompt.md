# Claude Code 用：GitHub Issue 一括作成プロンプト

> **アーカイブ済み（2026-07-07）**: 初期 Issue #1〜#14 の起票に使用済み。
> 今後の一括起票は `/requirements-to-issues` スキルを使うこと。

以下をそのまま Claude Code に貼り付けて使う。`docs/issues/memo-app-issues.md` を同じリポジトリに置いておくこと。

---

あなたはこのリポジトリの開発を支援するエンジニアです。
添付の `docs/issues/memo-app-issues.md` に記載された Issue 一覧（#1〜#14）を、`gh` コマンドを使って GitHub Issue として作成してください。

## 前提確認（作業前に実行）

1. `gh auth status` で認証済みか確認する。未認証ならその旨を報告して停止する。
2. `gh repo view --json nameWithOwner` で対象リポジトリを確認し、これから Issue を作成する旨を私に伝える。

## ラベルの準備

以下のラベルが無ければ `gh label create` で作成する（既存ならスキップ）。

- `foundation`（基盤）— 色: `0E8A16`
- `feature`（機能）— 色: `1D76DB`
- `test`（テスト）— 色: `5319E7`

種別とラベルの対応：基盤→`foundation` / 機能→`feature` / テスト→`test`。

## Issue 作成手順

`docs/issues/memo-app-issues.md` の各 Issue 本文（`## [#N] タイトル` から次の `## [#...]` の手前まで）を、その Issue の body としてそのまま使う。

1. **番号順（#1→#14）に**作成する。依存関係の参照を後から番号で貼れるようにするため。
2. 各 Issue の body は一時ファイルに書き出してから `--body-file` で渡す（本文に改行・チェックボックス・テーブルが含まれるため、`--body` でのインライン渡しは避ける）。

   ```bash
   gh issue create \
     --title "<タイトル>" \
     --body-file /tmp/issue_N.md \
     --label "<対応ラベル>"
   ```

3. 作成のたびに返ってくる Issue URL / 番号を控え、「マークダウン上の #N」→「実際に発行された GitHub Issue 番号」の対応表をメモしておく。
4. **本文中の依存参照を実番号に置換する。** マークダウンの `### 依存` に書かれた `#6, #3` などはあくまで一覧上の通し番号。GitHub が採番した実際の番号と異なる場合があるため、全 14 件を作成し終えた後に、対応表をもとに各 Issue 本文の依存番号を実 Issue 番号へ `gh issue edit <番号> --body-file ...` で更新する。
   - もしリポジトリの既存 Issue 番号が 0 件で、#1 から連番で採番される場合はこの置換は不要。その場合はスキップしてよい。

## 完了報告

最後に、作成した全 Issue を以下の表で報告してください。

| マークダウン番号 | GitHub Issue番号 | タイトル | URL |
|---|---|---|---|

## 注意

- `--body-file` に渡す一時ファイルは作成後に削除する。
- 「要決定」と書かれた箇所は本文にそのまま残す（消さない）。後で私が判断する。
- 不明点があれば作成を止めて確認すること。勝手に仕様を補完しない。
