# ストリーク表示・カレンダーヒートマップの追加

以下の内容を実装してください。

## 背景

「1日1新」は継続習慣の記録であり、「今日は書いたか」「何日続いているか」が一目でわかる可視化との相性が良い。習慣トラッカー系アプリでは、ひと目でわかるストリークやヒートマップが継続モチベーションの中核機能とされている。現状のアプリにはどちらも存在しない。

## 変更内容

1. GitHub のコントリビューショングラフのような**月表示のカレンダーヒートマップ**コンポーネントを作成する
   - データは各機能の `items` の `created_at` の日付集合からクライアント側で算出する（新しいクエリは不要）
   - 依存ライブラリは追加せず、CSS Grid または SVG で自前描画する
   - 実装前に `dataviz` スキルを読み、配色・アクセシビリティの指針に従う
2. **連続記録日数（ストリーク）**を算出する純粋関数を `app/utils/` に追加する（今日または昨日を起点に、記録のある日が連続している日数）
3. `/one-new` と `/topics` のページ上部に、ヒートマップとストリーク（例：「🔥 5日連続」）を表示する

## 対象範囲・制約

- 対象：新規コンポーネント（例：`app/components/StreakHeatmap.vue`）、`app/utils/` の新規純粋関数、`one-new` / `topics` ページ、関連テスト
- `/workout` への設置と月次サマリー（`ux-21-monthly-summary.md`）はスコープ外
- ヒートマップは当月（または直近の数週間）のみでよい。年間表示は不要

## 完了条件

- ストリーク算出関数の単体テスト（連続・途切れ・記録なし・同日複数件のケース）を追加する
- ヒートマップのコンポーネントテスト（日付集合→セル描画）を追加する
- `npm run test` と `npm run lint` が通る
