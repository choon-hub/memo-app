# 検証ループを「実アプリ駆動」まで伸ばす（UI 変更時のスクリーンショット確認）

以下の内容を実行してください。

## 背景

現状の検証は `npm run test` ＋ `npm run lint`（ローカルと CI）で、テストが通っても実際の画面で崩れているケースは検出できない。AI エージェントは「できたように見える」時点で止まりやすいため、UI 変更については「実アプリを起動して観察する」ところまでを標準の検証手順として明文化する。`/verify`・`/run` スキルが利用可能。

## 変更内容

1. AGENTS.md の「テスト方針」節に、UI に影響する変更の検証手順を 2〜3 行追記する：
   - UI（コンポーネント・ページ・CSS）に影響する変更では、テストに加えて `npm run dev` で実際に起動し、対象ページのスクリーンショットを撮って変更前の意図と比較する（`/verify`・`/run` スキルを利用する）
   - 差分がある場合は根本原因を修正する（エラーや警告の抑制で済ませない）
2. 追記はこの1点のみ。Stop hook による `npm run test && npm run lint` のゲート化は、現状 CI が同じ役割を担っているため実施しない（必要になった際の選択肢として、追記文には含めず作業報告にのみ言及する）

## 対象範囲・制約

- 対象：AGENTS.md のみ（CLAUDE.md は @AGENTS.md import のため変更不要）
- 追記は簡潔にする。「これを消すと Claude はミスするか？」の基準を満たす表現のみ残す

## 完了条件

- AGENTS.md の追記が完了し、`npm run lint`（prettier チェック）が通る
