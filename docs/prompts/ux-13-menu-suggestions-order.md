# 筋トレのメニュー候補チップを「直近使用順」にする

以下の内容を実装してください。

## 背景

`useWorkout.ts` の `getMenuCandidates` は `Set` 生成後の順序（＝データ取得順）で先頭5件を出すだけで、頻度順でも直近順でもない。よく使うメニューが候補に埋もれる可能性がある。

## 変更内容

1. `getMenuCandidates` を、該当カテゴリのレコードを `created_at` 降順（直近使用順）に並べたうえでメニュー名の重複を除き、上位5件を返すように変更する
2. ロジックが複雑になる場合は、純粋関数として `app/utils/` に切り出してもよい（テストしやすさを優先）

## 対象範囲・制約

- 対象：`app/composables/useWorkout.ts`（または切り出した utils）、関連テスト
- `menuSuggestions`（全メニューのアルファベット順候補）の挙動は変えない
- 使用頻度によるスコアリングはスコープ外。直近使用順のみでよい

## 完了条件

- 「直近に使ったメニューが先頭に来る」「重複が除かれる」「5件で打ち切られる」ことを検証するテストを追加する
- `npm run test` と `npm run lint` が通る

## 関連

- `ux-01-workout-query-optimization.md` は同じ関数を触る。両方実施する場合は ux-01 を先に行い、その実装を前提にする
