# フォームにインラインバリデーションメッセージを追加する

以下の内容を実装してください。

## 背景

各フォーム（`DailyNewForm.vue` / `TopicForm.vue` / `WorkoutForm.vue`）は `computed` の `isDisabled` で送信ボタンを無効化するのみで、ユーザーはボタンが押せない理由（未入力の項目、数値の不正など）を知る手段がない。`WorkoutForm` のバリデーションが最も複雑。

## 変更内容

1. 各フォームのフィールドごとに、無効な入力の理由を示すインラインメッセージを表示する
   - 例：「タイトルを入力してください」「回数は1以上の整数で入力してください」
2. 初期表示でいきなり全フィールドにエラーを出さないよう、フィールドの blur 後（touched）または送信試行後にのみ表示する
3. メッセージ要素には対応する input の `aria-describedby` を設定する
4. 3フォームでメッセージの見た目・表示条件のパターンを揃える

## 対象範囲・制約

- 対象：3つの Form コンポーネントと関連テスト
- バリデーションライブラリは導入しない。既存の `computed` ベースの判定を拡張する
- 送信ボタンの `disabled` 制御は現状のまま維持する

## 完了条件

- コンポーネントテストで「不正入力＋blur でメッセージが表示される」「有効入力でメッセージが消える」ことを検証する（少なくとも WorkoutForm は数値系のケースを含める）
- `npm run test` と `npm run lint` が通る
