# ダークモード対応

以下の内容を実装してください。

## 背景

`@media (prefers-color-scheme: dark)` の使用箇所がゼロで、`app/assets/global.css` や各コンポーネントの配色（`#f9f9f9`、`white` 等）はすべて固定のライトテーマ。毎日使う個人用アプリのため、夜間利用を考えるとダークモード対応の価値が高い。

## 変更内容

1. `global.css` に色トークンの CSS 変数を定義する（背景・カード背景・文字色・ボーダー・アクセント・エラー等。既存の `--nav-height` と同じ流儀）
2. 各コンポーネント・ページに直書きされている色をすべて CSS 変数参照に置き換える
3. `@media (prefers-color-scheme: dark)` で変数値をダーク配色に切り替える
4. コントラストは本文テキストで 4.5:1 以上を保つ

## 対象範囲・制約

- OS 設定への追従のみとし、アプリ内の手動テーマ切り替え UI はスコープ外
- レイアウト・余白などの構造には手を入れず、色の変数化と切り替えのみ行う

## 完了条件

- リポジトリ内に直書きの色指定（`#xxx` / 色名）が残っていないことを grep で確認する（CSS 変数定義部を除く）
- `npm run test` と `npm run lint` が通る
- `npm run dev` で起動し、ライト/ダーク両方の配色でスクリーンショットを撮って表示崩れ・視認性を確認する（`/run` スキルが利用可能）
