# 筋トレ：種目別の推移グラフ・自己ベスト（PR）表示

以下の内容を実装してください。

## 背景

現状の筋トレ機能はメニュー候補（直近使った種目名の補完）までは実装済みだが、「同じ種目の過去記録との比較」ができない。筋トレ記録アプリでは重量・回数の推移グラフと自己ベスト表示が長期的な成長実感の中核機能とされている。

## 変更内容

1. `/workout` ページに種目（メニュー名）を選ぶと、その種目の記録推移を表示するセクションを追加する
   - 横軸：日付、縦軸：重量（`intensity`）。各点に回数（`reps`）を添える
   - 依存ライブラリは追加せず、SVG で自前描画する
   - 実装前に `dataviz` スキルを読み、配色・軸・アクセシビリティの指針に従う
2. 選択種目の自己ベスト（最大 `intensity`。同値なら `reps` が多い方）を「PR: 60kg × 8回（2026-07-01）」の形式で表示する
3. データは既存の取得結果から算出する。グルーピング・PR 算出は純粋関数として `app/utils/` に切り出す

## 対象範囲・制約

- 対象：新規コンポーネント（例：`WorkoutProgressChart.vue`）、`app/utils/` の新規純粋関数、`workout/index.vue`、関連テスト
- 推定 1RM の計算・PR 通知はスコープ外
- 記録が1件以下の種目ではグラフを出さず、その旨の軽い表示に留める

## 完了条件

- グルーピング・PR 算出関数の単体テスト（複数種目・同値 PR・記録なしのケース）を追加する
- コンポーネントテストで「種目選択→グラフ描画」の基本動作を検証する
- `npm run test` と `npm run lint` が通る
