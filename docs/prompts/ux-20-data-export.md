# データエクスポート機能（JSON / CSV バックアップ）

以下の内容を実装してください。

## 背景

RLS 未設定・単一ユーザー運用という現状のセキュリティ前提では、万一の Supabase プロジェクト障害やキー漏洩時の復旧手段として、手元にバックアップを取れる手段の価値が高い。既存の全件取得ロジックを変換するだけなので実装コストも低い。

## 変更内容

1. 3テーブル（`daily_new` / `topics` / `workout_records`）の全件を取得し、JSON ファイルとしてダウンロードする「エクスポート」機能を追加する
   - 形式：`{ daily_new: [...], topics: [...], workout_records: [...] }` を1ファイルに（ファイル名に日付を含める。例：`memo-app-export-2026-07-09.json`）
   - あわせてテーブルごとの CSV ダウンロードも選べるとよい（JSON を優先し、CSV が過剰なら省略して報告する）
2. 導線は `AppHeader.vue` またはナビゲーション近くに1箇所置く（各ページへの個別設置はしない）
3. 取得・変換ロジックは composable（例：`useExport`）または `app/utils/` の純粋関数（変換部分）に分け、変換部分をテスト可能にする

## 対象範囲・制約

- 対象：新規 composable / utils、`AppHeader.vue`（または適切な共通 UI）、関連テスト
- インポート（リストア）機能はスコープ外
- 依存ライブラリは追加しない（Blob ＋ `URL.createObjectURL` で実装できる）

## 完了条件

- 変換関数（レコード配列 → JSON/CSV 文字列）の単体テストを追加する（空データ・カンマや改行を含む値のエスケープを含む）
- `npm run test` と `npm run lint` が通る
