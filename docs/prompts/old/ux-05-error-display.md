# エラー表示の改善（role="alert" ＋ 再試行 ＋ 閉じる）

以下の内容を実装してください。

## 背景

3ページとも `<div v-if="error" class="error">{{ error }}</div>` という素のテキスト表示のみ（スタイルは `app/assets/global.css` の `.error`）。再試行ボタンがなく、`role="alert"` / `aria-live` がないためスクリーンリーダーに通知されず、表示位置がページ最上部固定で消す手段もない。

## 変更内容

1. エラー表示を共通コンポーネント（例：`app/components/AppErrorAlert.vue`）に切り出す
   - `role="alert"` を付与し、スクリーンリーダーに通知されるようにする
   - 「再試行」ボタンを添え、クリックで再フェッチ（各ページの `fetchList` 相当）を実行する
   - 明示的な「閉じる」ボタンを付ける（自動消去のトーストまでは不要）
2. `one-new` / `topics` / `workout` の3ページのエラー表示を新コンポーネントに置き換える

## 対象範囲・制約

- 対象：新規共通コンポーネント、3ページ（`app/pages/*/index.vue`）、`global.css` の `.error` スタイル、関連テスト
- トースト通知システムの導入はスコープ外。既存のエラー表示位置（ページ上部）のままでよい
- composable 側のエラーハンドリング（`error.value` へのセット）は変えない

## 完了条件

- コンポーネントテストで「エラーメッセージの表示」「`role="alert"` の付与」「再試行ボタンの emit（またはコールバック実行）」「閉じるで非表示」を検証する
- `npm run test` と `npm run lint` が通る
