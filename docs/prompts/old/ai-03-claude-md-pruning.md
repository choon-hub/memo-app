# CLAUDE.md / AGENTS.md の剪定と compaction 指示の追加

以下の内容を実行してください。

## 背景

肥大化した CLAUDE.md / AGENTS.md は重要なルールがノイズに埋もれ、指示が守られにくくなる。公式ベストプラクティスは「1行ごとに『これを消すと Claude はミスするか？』を自問し、しないなら削る」「Claude が指示なしでも正しくやれる行は削除するか hook に変換する」を推奨している。また、コンテキスト圧縮（compact）時に重要情報が落ちないよう、圧縮時の保持指示を CLAUDE.md に書ける。

## 手順

1. CLAUDE.md と AGENTS.md を1行ずつ評価する。判定基準：
   - **残す**：推測不能なコマンド、デフォルトと異なる規約、環境の癖、非自明な設計判断（例：`useSupabase()` がシングルトンである理由）
   - **削除候補**：コードを読めば分かること、標準的な言語規約・自明な一般論、Claude が指示なしでも正しくやれること、実装変更で古くなった記述
2. 削除候補を「行の内容／削除してよいと判断した理由」の一覧としてユーザーに提示し、承認を得てから削除する（勝手に削除しない）
3. CLAUDE.md に compaction 指示を1行追加する：
   > compact 時は変更ファイル一覧・実行したテストコマンド・未解決の受け入れ条件を保持すること
4. 変更後、`npm run lint`（prettier チェック）を通す

## 制約

- AGENTS.md が正本・CLAUDE.md は import ＋ Claude 固有補足という現行構成は維持する
- 剪定は削除のみとし、新しいルールの追加は compaction 指示の1行に限る
- この作業は月次などで繰り返し実施できる（実施日と削った行数を最後に報告する）
