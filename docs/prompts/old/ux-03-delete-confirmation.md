# 削除操作に確認ステップを追加する

以下の内容を実装してください。

## 背景

`app/components/DailyNewList.vue` の削除ボタンは押した瞬間に `emit('remove', item.id)` を呼び、確認ダイアログが存在しない。誤タップでデータが即座に消える。

## 変更内容

1. `DailyNewList.vue` の削除ボタンに、インラインの2段階確認を追加する
   - 1回目のクリックで「削除しますか？」の確認表示（「削除する」「キャンセル」）に切り替わる
   - 「削除する」で初めて `emit('remove', item.id)` を呼ぶ。「キャンセル」または他アイテムの操作で元に戻る
2. ブラウザ標準の `confirm()` は使わず、既存のマークアップ・CSS クラスの流儀に合わせたインライン UI にする（スマホでの誤タップ防止が目的のため、見た目は控えめでよい）

## 対象範囲・制約

- 対象：`app/components/DailyNewList.vue` と `app/components/__tests__/` の対応テスト
- 現時点で削除機能があるのは一日一新のみ。今後トピックに削除を追加する場合（`ux-12-topics-delete.md`）も同じパターンを踏襲する
- Undo（元に戻す）トーストはスコープ外とする

## 完了条件

- コンポーネントテストで「削除ボタン1回のクリックでは `remove` が emit されず、確認後のクリックで emit される」ことを検証する
- `npm run test` と `npm run lint` が通る
